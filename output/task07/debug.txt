Potential Runtime Warnings
--------------------------
1. Quaternion slerp edge case: When source and destination quaternions are
   exactly opposite, slerp may take the long path. Mitigated by ensuring
   quaternion continuity in pose definitions.

2. useFrame timing: state.clock.getDelta() returns time since last frame.
   In blendState, we accumulate elapsed time manually. This is correct
   but could drift if frames are dropped. Acceptable for MVP.

3. Gloss queue draining: If dequeueGloss is called while blending is in
   progress, the new gloss will be ignored until phase returns to 'idle'.
   This is intentional to prevent animation jank.

Debugging Tips
--------------
1. If avatar doesn't animate:
   - Check appStore.currentGloss is being set
   - Verify glossQueue has items
   - Ensure blendState.phase is 'idle' before triggering

2. If animation jitters:
   - Check quaternion continuity in islAnimationMap.js
   - Verify SLERP is being called (not direct assignment)
   - Check for NaN values in pose definitions

3. If NMMs don't apply:
   - Verify sentenceType is set in appStore
   - Check getNMMForSentenceType returns non-null
   - Ensure NMM pose merges with gloss pose correctly

4. Performance profiling:
   - Use React DevTools Profiler to check HubAvatar re-renders
   - Monitor useFrame execution time in Chrome DevTools
   - Check GPU memory usage in Performance panel
